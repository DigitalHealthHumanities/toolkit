For our lesson on network graphs and Gephi, we taught learners how to import data from a Microsoft Excel export (a .csv file) directly to Gephi. We did this for two reasons: 1) we wanted to focus on the data categories required to make Gephi operate, and 2) we wanted to lay a foundation about how workflows can function in digital research. 

For many digital tools, especially open-source ones, it can be easier and quicker to develop a dataset outside that tool before importing it later in the process. While these tools often have an interface for data entry, they can be clunkier or slower than just working from a tabular data platform like Microsoft Excel or Google Sheets.  

The workflow we used in that Gephi lesson involved the creation of a pair of comma separated value (.csv) spreadsheets, which would be imported into Gephi when the network needed to be generated.  

For this lesson, we will begin to think about how a multi-step data process interfaces with traditional, written research. The goal of this lesson is to show how the processes of entering data and digital tools links up with other practices associated with research. 

For this hands-on activity, we will begin to link multi-step digital processes with the print-centered workflows we generated in [the last lesson](https://digitalhealthhumanities.github.io/toolkit/lessons/projectmanagement/digitalworkflows/3). Like in that lesson, we will be asking you to draw out workflow graphs either on a piece of scratch paper, or on a digital platform like Microsoft Whiteboard.

Now that we have a sense of how Gephi is exporting data for our written, we should ask, how exactly do we enter and shape the data in such a manner so that it can be read by the network graphing platform? 

As we learned in the introduction to Microsoft Excel, it is not feasible to handwrite .csv files. Instead we’ll need a spreadsheet software, like Microsoft Excel, from which we’ll export. We also will need to be working with some sort of data, but we are going to ignore that part of the workflow for right now. (We will address that issue in more detail in the next section.) 

Add Microsoft Excel to your sketched workflow. It should connect to Gephi in some way, but it might also connect elsewhere in the workflow. If you come from a quantitative field, or plan to include quantitative data in the form of charts or graphs, be sure to include an “Exported Figures" box.
